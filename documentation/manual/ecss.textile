h1. Variables support in CSS files

This module allow you to use variables in the CSS files. It is very helpful when you work with large CSS files, to avoid to repeat yourself with color, fonts, etc ...

h2. <a>Enable this module for your application</a>

In any application, just add this line to the **conf/application.conf** file :

bc. module.ecss=${play.path}/modules/ecss

At application start time, you should see this message : 

bc. 00:21:45,192 INFO  ~ Module ecss is available (.../play/modules/ecss)

h2. <a>Define variables in the CSS files</a>

Just declare CSS variables in top of your CSS files and reuse them latter in the same file. 

Like this :

bc. $mainColor: #c00; $secondaryColor: green;
$font: verdana, sans-serif;
 
body {
   color: $mainColor;
   font-family: $font;
}
 
a {
   color: $mainColor;
   font-family: $font;
}
 
em {
   color: $secondaryColor
}

p(note). **Warning**
 
The ''ecss'' module just enable variables in static CSS files. The CSS files will not be parsed for each request.
 
If you want to enable dynamic CSS variables depending of your application context, it is another story... You can for example generate the stylesheet from a play Controller action, or define some CSS rules in your main HTML template.
